# 🚖 Ride Pulse – Cab Booking Data Analysis (SQL)

## 🔍 Project Overview

This project analyzes cab booking data using SQL to uncover demand trends, peak hours, and customer behavior patterns.

## 🎯 Objective

* Identify peak ride demand hours
* Analyze customer booking patterns
* Evaluate driver utilization and route performance

## 🛠 Tools Used

* SQL (MySQL / PostgreSQL)
* Joins, Aggregations, Subqueries
* Data Analysis Queries

## 📊 Dataset

* 5,000+ ride records including:

  * Ride time
  * Pickup & drop locations
  * Driver details
  * Fare amount

## 📈 Key Insights

* Peak demand observed during morning and evening hours
* Certain routes generate higher revenue consistently
* Driver utilization varies significantly across time slots
* Customer booking frequency shows pattern-based behavior

## 📂 Files Included

* ride_pulse_queries.sql
* dataset.csv

## 🧠 SQL Concepts Used

* INNER JOIN, LEFT JOIN
* GROUP BY, ORDER BY
* Subqueries
* Aggregate Functions (SUM, AVG, COUNT)

## 🚀 Skills Demonstrated

* Data Analysis using SQL
* Query Optimization
* Business Insights Extraction

## 💡 Conclusion

The analysis helps improve driver allocation, optimize routes, and enhance customer experience through data-driven decision-making.
